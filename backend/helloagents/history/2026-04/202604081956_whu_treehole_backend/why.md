# 变更提案: 武大树洞后端实现

## 需求背景
当前项目已具备前端界面，但数据完全来自本地常量与浏览器存储，无法支撑真实查询、发帖、互动和消息场景。需要补齐基于 Java Spring Boot、MySQL、MyBatis 与 Redis 的后端服务。

## 产品分析

### 目标用户与场景
- **用户群体:** 武大树洞前端的开发者与后续真实使用者
- **使用场景:** 浏览首页、进入话题页、查看校友圈、查看个人主页、发送站内消息
- **核心痛点:** 前端无法对接真实后端，缺少统一接口、数据库和缓存体系

### 价值主张与成功指标
- **价值主张:** 通过结构清晰的多模块后端，快速完成前后端联调和后续扩展
- **成功指标:** 后端可独立启动；核心页面有稳定 API；发帖、点赞、收藏、关注和消息发送可用

### 人文关怀
当前版本默认支持匿名发帖，避免在演示阶段暴露用户隐私；正式上线前需补充鉴权与权限控制。

## 变更内容
1. 新建 Maven 多模块 Spring Boot 后端工程。
2. 建立 MySQL 数据模型与初始化种子数据。
3. 提供首页、话题页、校友圈、个人主页与消息中心接口。
4. 提供帖子互动、关注切换与消息发送接口。
5. 接入 Redis 缓存热点页面聚合结果。

## 影响范围
- **模块:** common、domain、infra、server
- **文件:** `backend/**`
- **API:** `/api/v1/pages/*`、`/api/v1/posts/*`、`/api/v1/conversations/*`
- **数据:** MySQL 表结构、Redis 页面缓存

## 核心场景

### 需求: 页面聚合接口
**模块:** server
为现有前端提供页面级查询接口，减少前端拆分请求成本。

#### 场景: 首页与校友圈查询
用户进入首页或校友圈时：
- 能拿到完整页面所需的数据
- 支持按话题或关键词过滤帖子

### 需求: 内容互动
**模块:** server
支持当前前端已存在的发帖、点赞、收藏和关注操作。

#### 场景: 发帖与互动回写
用户提交帖子或切换点赞/收藏/关注时：
- 数据写入数据库
- 返回可直接更新前端的最新状态

### 需求: 消息中心
**模块:** server
支持查看会话、读取消息、发送新消息和清空未读数。

#### 场景: 站内消息发送
用户在个人页发送消息时：
- 新消息写入数据库
- 会话摘要和排序时间同步更新

## 风险评估
- **风险:** 当前前端尚未接入鉴权，后端只能以演示用户模式提供能力
- **缓解:** 在接口与文档中明确演示用户约束，预留后续鉴权扩展空间
