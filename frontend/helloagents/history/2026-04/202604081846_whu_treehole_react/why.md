# 变更提案: 武大树洞 React 化重构

## 需求背景
当前仓库中只有 `index.html`、`topic.html`、`campusFriends.html`、`person.html` 四个独立静态页面。它们视觉风格接近，但代码完全分散，没有共享布局、没有真实路由，也缺少统一的数据与交互层，后续难以继续演化成完整站点。

## 产品分析

### 目标用户与场景
- **用户群体:** 武汉大学在校学生、毕业校友、希望在校园语境下交流的人
- **使用场景:** 浏览校园树洞、查看热门话题、进入校友圈、管理个人资料和消息
- **核心痛点:** 页面孤立、体验割裂、无法持续扩展交互

### 价值主张与成功指标
- **价值主张:** 用统一的前端架构把树洞、话题、校友圈和个人主页整合为一个可继续迭代的社区站点
- **成功指标:** 四个页面可通过路由互相切换；存在统一导航；至少具备发布、点赞、收藏、关注、消息发送等基础交互

### 人文关怀
默认保留匿名表达语境，不引入真实身份校验；当前版本不采集个人隐私数据，交互仅保存在浏览器本地。

## 变更内容
1. 使用 React + TypeScript + Vite 重建前端工程结构
2. 将四个静态页面迁移为首页、话题、校友圈、个人主页四个路由页面
3. 抽取共享布局、帖子卡片、图标、发布弹层与统一样式系统
4. 建立本地状态层，支持发布、点赞、收藏、关注和消息发送

## 影响范围
- **模块:** app-shell、home-feed、topic-explorer、alumni-circle、profile-center
- **文件:** `index.html`、`package.json`、`src/**`、`helloagents/**`
- **API:** 无外部 API 变更
- **数据:** 新增前端本地数据模型与 `localStorage` 快照

## 核心场景

### 需求: 首页树洞体验
**模块:** home-feed
用户进入站点后能够浏览校园树洞、按话题过滤内容并进行基础互动。

#### 场景: 浏览首页内容
用户打开首页时，能看到主视觉区、话题入口、帖子信息流和热议榜。
- 可以基于搜索词或话题筛选帖子
- 可以点赞、收藏帖子并看到统计变化

### 需求: 话题入口聚合
**模块:** topic-explorer
用户从话题页进入不同内容场景时，应保持路径清晰。

#### 场景: 进入指定话题
用户在话题页点击分类卡片后，跳转到首页或校友圈对应筛选状态。
- 跳转后目标页面保留话题条件
- 话题页可展示热门排行和标签组合

### 需求: 校友圈内容浏览
**模块:** alumni-circle
用户可以查看校友动态、内推和关系推荐。

#### 场景: 浏览校友圈
用户打开校友圈时，能按标签和搜索词筛选帖子。
- 可以关注推荐校友
- 发布到校友圈的内容应在对应页面展示

### 需求: 个人主页与消息
**模块:** profile-center
用户可以查看自己的资料、帖子、收藏，以及与他人的私信会话。

#### 场景: 发送站内消息
用户进入个人主页的消息中心时，可以选择会话并发送一条新消息。
- 消息区即时追加新内容
- 会话列表摘要与时间同步更新

## 风险评估
- **风险:** 由静态页直接迁移到 SPA 时容易引入结构过度集中或样式失真
- **缓解:** 先建立共享数据模型和布局骨架，再逐页迁移视觉细节，最后统一验证构建
