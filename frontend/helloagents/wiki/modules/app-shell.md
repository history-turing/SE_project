# app-shell

## 目的
统一站点导航、页脚、移动端入口和全局发布弹层。

## 模块概述
- **职责:** 承载所有页面的共享布局和全局交互
- **状态:** 🚧开发中
- **最后更新:** 2026-04-08

## 规范

### 需求: 统一站点外壳
**模块:** app-shell
将原本分散的静态页收敛为共享导航与页脚，避免跨页面重复维护。

#### 场景: 用户在任意页面切换主导航
- 当前路由高亮正确
- 移动端底部导航与桌面端导航保持一致

#### 场景: 用户从任意页面发帖
- 点击“发布”即可打开弹层
- 发布后首页和个人页内容同步刷新

### 需求: 远程部署入口统一
**模块:** app-shell
生产环境中，站点应由同域入口承载静态资源与 API 访问，避免浏览器直接依赖开发代理。

#### 场景: 用户访问远程部署站点
- 页面静态资源由 Nginx 提供
- 前端路由刷新后仍能回到 `index.html`
- `/api/v1` 请求通过同域代理转发到后端容器
