# 项目技术约定

---

## 技术栈
- **核心:** TypeScript / React / Vite
- **路由:** React Router
- **样式:** 原生 CSS + CSS 变量
- **状态:** React Context + 本地组件状态

---

## 开发约定
- **代码规范:** 组件与页面按职责拆分，避免把结构、样式、状态堆叠在单文件中
- **命名约定:** 组件与类型使用 PascalCase，变量与函数使用 camelCase，常量使用语义化英文
- **页面原则:** 通过共享布局、共享数据模型和共享交互减少重复

---

## 错误与日志
- **策略:** 当前为纯前端演示站点，使用安全默认值与空态兜底，不引入运行时日志系统
- **数据持久化:** 交互状态优先写入 `localStorage`，避免依赖外部服务

---

## 测试与流程
- **测试:** 当前以 `vite build` 构建校验为主，后续可按模块补充组件测试
- **提交:** 变更完成后同步更新 `helloagents/` 知识库与历史方案包

---

## 部署约定
- 生产环境使用多阶段 Docker 镜像构建，最终由 Nginx 承载 `dist/` 目录。
- 浏览器请求 `/api/v1` 时由 Nginx 反向代理到后端容器，默认不再依赖 Vite 开发代理。
- 远程部署默认使用同域访问，`VITE_API_BASE_URL` 维持 `/api/v1`。
